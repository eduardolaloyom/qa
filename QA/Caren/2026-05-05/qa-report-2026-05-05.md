### CAREN-QA-001: assignedSellers falla — sin badges de estado en comercios
- **Severidad:** P1 · Bug de integración
- **Área:** Sync / API
- **Descripción:** El request `assignedSellers` falla con 0 requests durante cada sync. Como resultado, la lista de comercios no muestra badges de estado (Disponible / Bloqueado). Esto impide identificar qué comercios están bloqueados y bloquea la validación de 4 flows (03, 05, 05b, parciales 09 y 10).
- **Evidencia:** Capturado en logcat previo (QA/Caren/2026-04-23/logcat-sync2.log) — `assignedSellers: 0 requests` en todas las syncs monitoreadas.
- **Impacto:** La feature de bloqueo crediticio no es verificable en staging. Vendedor no puede distinguir comercios disponibles de bloqueados.
- **Estado:** Abierto

### CAREN-QA-002: Formulario "Crear Comercio" abre pero no carga campos
- **Severidad:** P2 · Features / Bug
- **Área:** Creación de comercios (Sección 2.2 del alcance)
- **Descripción:** Al tocar "Crear Comercio", se abre la pantalla "Crear nuevo comercio" con el texto "Ingresa datos de comercio" pero sin campos de formulario — solo un spinner y el botón "Enviar formulario" vacío. La pantalla está en blanco. Causa a investigar: la API que entrega la definición del formulario puede estar retornando vacío, con error, o hay un bug de renderizado en frontend.
- **Pasos:** 1. Login vendedor Caren en staging. 2. Ver listado de comercios. 3. Tocar "Crear Comercio". 4. Pantalla abre con formulario vacío — sin campos.
- **Evidencia:** Screenshot manual — pantalla "Crear nuevo comercio" con spinner y sin campos (2026-05-05).
- **Referencia alcance:** Sección 2.2 — "Solicitud de creación de comercios: API que recibe las solicitudes."
- **Estado:** Abierto — requiere debug de API o frontend
