<objective>
Make `tools/live-reporter.js` write to `public/live.json` atomically so concurrent readers (the dashboard browser) and overlapping playwright processes never see partial/invalid JSON.

Purpose: REQ-06 — the dashboard polls live.json every 3 seconds during test runs. A non-atomic write can produce truncated JSON that causes the live panel to flicker/hide. The fix uses POSIX `rename(2)` atomicity guarantee: write to a `.tmp` file, then rename to the final path in a single syscall.

Output: Fixed `_save()` method in live-reporter.js, `.gitignore` updated, `run-live.sh` EXIT trap updated.
</objective>

2. Replace the `_save()` method body (lines 63-67) to use write-then-rename:

```javascript
// BEFORE:
_save() {
  try {
    fs.writeFileSync(OUTPUT, JSON.stringify(this.state, null, 2));
  } catch (e) { /* ignore */ }
  this._pushToGitHub(false);
}

// AFTER:
_save() {
  try {
    fs.writeFileSync(TMP_OUTPUT, JSON.stringify(this.state, null, 2));
    fs.renameSync(TMP_OUTPUT, OUTPUT);
  } catch (e) { /* ignore */ }
  this._pushToGitHub(false);
}
```

Why this works: POSIX `rename(2)` atomically swaps directory entries in a single kernel syscall. Readers always see either the old complete file or the new complete file, never a partial write. The `.tmp` file is in the same directory as the final file to guarantee same-filesystem (avoids EXDEV errors).

Do NOT add any npm dependencies. Do NOT change `onBegin`, `onTestBegin`, `onTestEnd`, `onEnd`, or `_pushToGitHub` methods — only `_save()` changes plus the new constant.
