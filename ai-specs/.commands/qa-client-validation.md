# QA Client Validation

Execute full QA suite for a client: MongoDB extract → sync clients → run Playwright → run Maestro → generate consolidated report.

## Usage

```
/qa-client-validation Codelpa prod
/qa-client-validation Surtiventas staging
/qa-client-validation "Seis Sur" prod --skip-maestro
```

Parameters:
- `{CLIENT}` — Client name (must match slug in qa-matrix.json)
- `{ENV}` — `prod` (youorder.me) or `staging` (solopide.me)
- `--skip-maestro` — Skip Maestro flows (if device/emulator not available)

## Steps

1. **Adopt QA Coordinator role** (see `ai-specs/.agents/qa-coordinator.md`)

2. **Verificar scope-gap-report**
   - Buscar `QA/{CLIENT}/*/scope-gap-report.md` (el más reciente)
   - Si no existe → ejecutar `/qa-scope-check {CLIENT}` para generarlo antes de continuar
   - Si existe → revisar la sección "Gaps — Sin Cobertura" y la sección "Desarrollos Custom"
     - Si hay gaps P0 (desarrollos custom sin test) → indicar cuáles son y preguntar si continuar sin ellos o crearlos primero
     - Si hay gaps P1 → anotarlos como "cobertura pendiente" en el reporte final

3. **Verificar checklist pre-producción**
   - Buscar `QA/{CLIENT}/*/checklist-preproduccion.md` (el más reciente)
   - Si **no existe** → indicar: _"Ejecutar primero `/qa-plan-client {CLIENT}` para generar el checklist"_ y detener
   - Si existe → leer el veredicto final:
     - `LISTO` → continuar
     - `CON CONDICIONES` → mostrar los items pendientes, confirmar con el usuario si continuar
     - `BLOQUEADO` o sin veredicto escrito → mostrar los items en estado BLOQUEADO o ☐, detener y escalar al dueño correspondiente (Tech/Analytics)
   - Verificar también: `.env` tiene credenciales para el cliente (`{SLUG}_COMMERCE_EMAIL`, `{SLUG}_COMMERCE_PASSWORD`)

3. **Extract MongoDB config (if stale)**
   ```bash
   # Staging (solopide.me — default day-to-day)
   python3 data/mongo-extractor.py --env staging --output data/qa-matrix-staging.json

   # Prod (youorder.me)
   python3 data/mongo-extractor.py --env production
   ```
   Extracts from: yom-stores, yom-production, yom-promotions, b2b-marketing

4. **Sync clients.ts**
   ```bash
   # Staging (most common — solopide.me clients)
   python3 tools/sync-clients.py --input data/qa-matrix-staging.json

   # Prod (youorder.me clients — run extractor first)
   python3 tools/sync-clients.py
   ```
   Regenerates `tests/e2e/fixtures/clients.ts` from the specified matrix.
   **Note:** This file is AUTO-GENERATED — never edit manually.

5. **Run Playwright tests**
   ```bash
   npx playwright test --project=b2b -g "{CLIENT_SLUG}"
   ```
   Collect: pass/fail counts, failure reasons, screenshots

6. **Run Maestro flows (if device available)**
   ```bash
   maestro test tests/app/flows/
   ```
   Collect: flow results, timing, failures

7. **Run relevant checklists**
   - Identify which checklists apply based on client config (from qa-matrix.json):
     - Has Khipu? → `checklist-fintech-khipu.md`
     - Has ERP hooks? → `checklist-integraciones-erp.md`
     - Has custom pricing? → `checklist-pricing-engine.md`
   - Note: manual checklists require human execution

8. **Generate consolidated report**
   - Use `/report-qa {CLIENT} {DATE}` to generate full report
   - Save to: `QA/{CLIENT}/{DATE}/qa-validation-{DATE}.md`

## Quick Mode

For fast validation (Playwright only, no Maestro):
```bash
./tools/run-qa.sh {CLIENT}
```

## Expected Output

After running:
- `test-results/` — Raw Playwright results
- `playwright-report/` — HTML report
- `QA/{CLIENT}/{DATE}/` — QA report (generated by /report-qa)
- Console summary: pass rate, critical failures, recommendations

## Interpreting Results

### P0 (Stop everything)
- Auth broken — commerce can't login
- Checkout impossible — orders can't be created
- Payment gateway down

### P1 (Fix before release)
- Config validation failing — client-specific setting not visible
- Multi-client discrepancy — works for some clients but not others

### P2 (Fix this sprint)
- Assertion timeouts — performance or timing issue
- Maestro selector stale — UI changed, need to re-record

### P3 (Backlog)
- Cosmetic failures
- Non-critical paths
