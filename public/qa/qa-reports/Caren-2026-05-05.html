CAREN-QA-005 — blockedClientAlert en todos los comercios

Severidad: P1 · Bug UX

Descripción: Al entrar a cualquier comercio y tocar "Hacer pedido", aparece el popup "Este comercio se encuentra bloqueado por estado crediticio" con botón "Continuar". El vendedor debe descartar este popup en cada visita. Confirmado con screenshot en flow 06 — comercio ID 0010008876 (ARRIENDO DE MAQUINARIAS ACONCAGUA LTDA).

Pasos: 1. Login como vendedor Caren en staging. 2. Entrar a cualquier comercio. 3. Tocar "Hacer pedido". 4. Popup aparece inmediatamente.

Causa probable: Consecuencia del bug assignedSellers (CAREN-QA-001) — la app no distingue estados y trata todos los comercios como bloqueados.

CAREN-QA-001 — assignedSellers falla con 0 requests

Severidad: P1 · Bug de integración

Descripción: El request assignedSellers falla con 0 requests durante cada sync. La lista de comercios no muestra badges de estado (Disponible / Bloqueado), impidiendo identificar comercios bloqueados y bloqueando la validación de 4 flows (03, 05, 05b, parciales 09 y 10).

Evidencia: Capturado en logcat (QA/Caren/2026-04-23/logcat-sync2.log) — assignedSellers: 0 requests en todas las syncs monitoreadas.

Impacto: Feature de bloqueo crediticio no validable. Vendedor no puede distinguir comercios disponibles de bloqueados.

CAREN-QA-002 — "Crear Comercio" visible pero funcionalidad no operativa

Severidad: P2 · Features / Integración

Descripción: El botón "Crear Comercio" aparece en lista de comercios (enableCreateCommerce: true), pero al intentar crear un comercio la funcionalidad no opera. Causa probable: la API de Caren para solicitudes de creación de comercios (Sección 2.2 del alcance) no está expuesta en staging.

Referencia alcance: Sección 2.2 — "Solicitud de creación de comercios: API que recibe las solicitudes y se le puede preguntar por el estado de las solicitudes enviadas."

Acción requerida: Caren debe exponer la API en staging para validar el flujo end-to-end.

CAREN-QA-003 — Barra de búsqueda no encontrada

Severidad: P2 · UX

Descripción: Ningún tap (texto "Buscar/Search/Filtrar" ni coordenadas 85%,8% y 50%,8%) activó barra de búsqueda en el listado de comercios. Puede no estar implementada en esta versión o tener selector distinto.
